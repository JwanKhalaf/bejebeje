## ADDED Requirements

### Requirement: User entity SHALL have a Slug property
The `User` domain entity in `Bejebeje.Domain` SHALL have a `Slug` property of type `string`. The property is non-nullable after Migration 2.

#### Scenario: User entity includes Slug property
- **GIVEN** the `User` class in `Bejebeje.Domain`
- **WHEN** the class definition is inspected
- **THEN** a public `Slug` property of type `string` SHALL exist alongside the existing properties (`Id`, `CognitoUserId`, `Username`, etc.)

### Requirement: Database SHALL have a slug column on the users table with a unique index
The `users` table SHALL have a `slug` column (text, not null after Migration 2, unique). The EF Core configuration in `BbContext.OnModelCreating` SHALL include `entity.HasIndex(e => e.Slug).IsUnique().HasDatabaseName("ix_users_slug")`.

#### Scenario: EF Core configuration includes unique index on Slug
- **GIVEN** the `BbContext.OnModelCreating` method
- **WHEN** the `builder.Entity<User>` configuration block is inspected
- **THEN** it SHALL include a unique index on `Slug` with database name `ix_users_slug`

#### Scenario: Slug column exists in users table after migrations
- **GIVEN** both database migrations have been applied
- **WHEN** the `users` table schema is inspected
- **THEN** a `slug` column of type text SHALL exist with a NOT NULL constraint and a unique index named `ix_users_slug`

### Requirement: Slugs SHALL be generated using NormalizeStringForUrl with collision resolution
Slugs SHALL be generated by applying `NormalizeStringForUrl()` (from `Bejebeje.Common.Extensions.StringExtensions`) to the trimmed username. If the result is empty, the fallback `user-{cognitoUserId[..Min(8, len)]}` SHALL be used. If the candidate slug already exists for a different user, a numeric suffix SHALL be appended (`-2`, `-3`, etc.) until unique.

#### Scenario: Standard slug generation from username with spaces
- **GIVEN** a username `"ali fm "`
- **WHEN** a slug is generated
- **THEN** the slug SHALL be `"ali-fm"` (trimmed, then NormalizeStringForUrl applied)

#### Scenario: Slug generation from username with mixed case and multiple words
- **GIVEN** a username `"Basnar Muhammed Ali"`
- **WHEN** a slug is generated
- **THEN** the slug SHALL be `"basnar-muhammed-ali"`

#### Scenario: Collision resolution appends numeric suffix
- **GIVEN** a username that generates slug `"kardox"` AND a different user already has slug `"kardox"`
- **WHEN** the slug is generated with collision resolution
- **THEN** the slug SHALL be `"kardox-2"`

#### Scenario: Multiple collisions increment suffix
- **GIVEN** a username that generates slug `"kardox"` AND users already hold slugs `"kardox"` and `"kardox-2"`
- **WHEN** the slug is generated with collision resolution
- **THEN** the slug SHALL be `"kardox-3"`

#### Scenario: Own slug does not count as collision during regeneration
- **GIVEN** an existing user with slug `"ali-fm"` whose username changes but still generates slug `"ali-fm"`
- **WHEN** slug collision resolution runs for that user
- **THEN** the user's own existing slug SHALL NOT count as a collision, and the slug SHALL remain `"ali-fm"`

#### Scenario: Empty slug fallback when NormalizeStringForUrl returns empty string
- **GIVEN** a username consisting entirely of characters stripped by NormalizeStringForUrl (e.g. only emoji)
- **WHEN** a slug is generated
- **THEN** the base slug SHALL be `"user-{cognitoUserId[..Min(8, cognitoUserId.Length)]}"` with collision resolution applied

### Requirement: Slugs SHALL be computed at write time only
Slugs SHALL be stored and indexed. They SHALL be computed at user creation or username change, never at read time.

#### Scenario: Slug is stored on user creation
- **GIVEN** a new user is created via `EnsureUserExistsAsync`
- **WHEN** the user record is saved
- **THEN** the `Slug` column SHALL contain the generated slug value

#### Scenario: Profile lookup uses stored slug without recomputation
- **GIVEN** a user with a stored slug
- **WHEN** a profile lookup is performed
- **THEN** the query SHALL match against the stored `slug` column, not recompute the slug from the username
